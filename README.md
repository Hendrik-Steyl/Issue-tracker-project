# Issue-tracker-project
PROJECT OVERVIEW
----------------
BugTracker is a streamlined, web-based issue tracking system designed 
for development teams. It features a modern, responsive interface for managing 
software bugs, tracking their priority, and assigning them to team members. 

This project was developed as a collaborative group effort to demonstrate 
full-stack frontend development practices using HTML5, CSS3 (including 
Bootstrap 5), and Vanilla JavaScript.

CORE FEATURES
-------------
* Secure Login System: Admin-only access with session management.
* Interactive Dashboard: Three-column layout for optimized workflow.
    - Column 1: Creation of new issues and admin seeding tools.
    - Column 2: Searchable and filterable list of all active issues.
    - Column 3: Real-time JSON data preview for storage monitoring.
* Automated Logic: Built-in overdue tracking for expired target dates.
* Issue Management: Comprehensive Create, Read, and Update (CRUD) functionality 
  via interactive modals.
* Local Storage Persistence: Data remains available across browser sessions 
  without requiring a backend database.

TECHNICAL SPECIFICATIONS
------------------------
* Frontend Architecture: HTML5, Bootstrap 5.3.0, and Custom CSS3.
* Logic Engine: Vanilla JavaScript (ES6+) using strict mode.
* Data Persistence: Browser localStorage and sessionStorage.
* Visuals: Custom SVG/Emoji-based iconography and CSS Grid/Flexbox layouts.

FILE STRUCTURE
--------------
* index.html - Core application structure and UI components.
* style.css  - Custom dark-themed UI/UX design and responsive breakpoints.
* script.js  - Application logic, data models, and storage management.
